Mathematical Core of Computation: Input, Function, and Output Relationships

1. Anatomy of the Computational Triad Sec. 01

Before approaching algorithmic formalisms like Supervised or Reinforcement learning, computation must be stripped down to its mathematical scaffolding. Every computational entity is an interplay between three pillars:

I. The Input (\(X\))

The measured vector variable space or environmental raw state. In mathematical mappings, it represents individual points drawn from an independent domain space matrix \(\mathcal{X}\).

II. The Function (\(f\))

The deterministic structure, operator matrix, or mapping rule system. It dictates exactly how input states are processed, shifted, or projected into dependent vector bounds.

III. The Output (\(Y\))

The dependent variable result, class label category, or system transformation target. It occupies the codomain range target space \(\mathcal{Y}\).

2. Case 1: Deductive Computation (Forward Execution) Sec. 02

In classic software engineering, standard algorithmic computation solves for the unknown variable Output (\(Y\)) given a known Input (\(X\)) and a fully transparent, explicit, predefined Function (\(f\)).

Forward Execution Architecture
Input (\(X\))
→
Known Rule Function
\(f(X)\)
→
Output (\(Y\))

Mathematical Formalism & Formulation

If the mapping rule is known explicitly as an algebraic matrix expression, computation reduces to basic forward evaluation:

$$Y = f(X)$$

For a multi-feature system where the function maps a weight vector transposed across inputs plus a bias parameter, computing the result takes the following form:

$$Y = \mathbf{W}^T X + b$$
Concrete Engineering Example

Scenario: Calculating a student's final marks based on a predefined absolute weight criteria rule.

  • Input Vector components: Exam marks \(x_1 = 85\), Assignment marks \(x_2 = 90\).
  • Known Transformation Function: \(f(x_1, x_2) = 0.70x_1 + 0.30x_2\)
  • Computed Output Scalar: \(Y = 0.70(85) + 0.30(90) = 59.5 + 27.0 = 86.5\)

3. Case 2: Inductive Computation (System Identification) Sec. 03

In modern Artificial Intelligence and empirical modeling, we observe the environmental reactions (Input, \(X\)) and match them against recorded targets (Output, \(Y\)). The inner structural transformation rule—the Function (\(f\))—remains completely hidden inside a black box. Our task is to reverse-engineer it via System Identification.

System/Function Identification Architecture
Input (\(X\))
→
Hidden Unknown Black-Box System Matrix
→
Output (\(Y\))
↓
Goal: Derive Function Rule Estimate (\(\hat{f}\))

Mathematical Reverse-Engineering Formulation

Because an infinite number of equations could technically satisfy a single coordinate mapping, we collect a dense historical observation matrix repository \(\mathcal{D}\) matching inputs to targets:

$$\mathcal{D} = \left\{ (x_1, y_1), (x_2, y_2), \dots, (x_N, y_N) \right\} \subset \mathcal{X} \times \mathcal{Y}$$

We configure an optimization engine to search within a defined hypothesis space framework \(\mathcal{H}\) to discover an optimized functional representation setting \(\hat{f}\) that minimizes total empirical loss:

$$\hat{f} = \arg\min_{f \in \mathcal{H}} \frac{1}{N}\sum_{i=1}^{N} \mathcal{L}\Big( y_i, f(x_i) \Big)$$
Concrete Reverse-Engineering Example

Scenario: Discovering the linear degradation property of an engineering system under stress over time.

  • Sample Inputs Observed: Run hours \(x \in \{100, 200, 300, 400\}\).
  • Sample Target Outputs Recorded: Efficiency measures \(y \in \{95\%, 90\%, 85\%, 80\%\} \equiv \{0.95, 0.90, 0.85, 0.80\}\).
  • Identified System Function: The training loop minimizes the residual squared error to isolate the uniform rate parameters, yielding: \(\hat{f}(x) = 1.00 - 0.0005x\).

4. Scaffolding Foundations for Core Machine Learning Paradigms Sec. 04

This structural core directly dictates the fundamental operational behavior of the upcoming core machine learning algorithm families:

Algorithm Family Class Input Condition Vector (\(X\)) Target Output Class (\(Y\)) The Learning Task Strategy (\(f\))
Supervised Learning Fully Known Data Points Fully Provided Labeled Target Ground-Truth Direct structural approximation of \(Y = f(X)\) using optimization cost functions.
Self-Supervised Learning Unlabeled Raw Input Stream (\(X_{raw}\)) Self-Generated Fake Targets (\(X_{masked}\) or \(X_{next}\)) The system automatically splits or hides parts of the input to create an internal artificial target. It tracks \(f(X_{visible}) = X_{hidden}\) (e.g., predicting the next word in a text line) to build deep baseline context mappings without manual human labels.
Unsupervised Learning Fully Known Data Points Completely Missing / Not Provided Derives latent structural hidden groups or sorting relations directly from geometric vector patterns.
Reinforcement Learning Environmental State Observations Delayed Reward Signal Responses Optimizes a policy trajectory mapping matrix through iterative interactive feedback trials.
